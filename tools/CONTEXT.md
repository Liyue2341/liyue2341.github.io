## What it is
Yue Li's (李越) personal academic homepage. al-folio–inspired, clean academic.
**Pure static: hand-written HTML/CSS/vanilla JS, NO build step, NO framework.**

## Architecture
- Pages: `index.html` (about + news + selected pubs + notebook + ...), `publications.html`,
  `blog.html`, `post.html?slug=…`, `404.html`. Shared header/footer is duplicated in each.
- All logic in `assets/js/app.js` (one file). Pages opt in via `<body data-page="…">`.
- **Content is data, rendered client-side**: `data/*.json` + `blog/<slug>/index.md`.
  `app.js` fetches and renders them. The JSON is portable, so another site can
  consume the same files instead of duplicating them.
- Styling: `assets/css/style.css`, CSS variables, light/dark via `[data-theme]` +
  `prefers-color-scheme` + a FOUC-guard inline script in each `<head>`.
- Deploy: push to `Liyue2341/liyue2341.github.io`, GitHub Pages serves at the domain
  root. `.nojekyll` present. Paths are RELATIVE (work at root; also survived an
  earlier org/sub-path plan). No CI.

## Conventions (don't break these)
- **Publications** (`data/publications.json`): one object per paper. Author roles in
  the `authors` string: trailing `*` = corresponding; `**…**` = co-first (entry shows
  `(co-first)` when `**Yue Li**` is wrapped); `Yue Li` auto-highlighted; BibTeX strips
  markers. `highlight:true` → shown on home. `image`/`pdf` are repo-relative or http.
- **Blog**: one folder per post — `blog/<slug>/index.md` + its images in the same
  folder, referenced `![](fig.jpg)`. `app.js` rewrites those to `blog/<slug>/…`.
  `data/blog.json` holds metadata (slug/title/date/summary/tags).
- **Thumbnails**: transparent PNG (white bg removed, border-connected so no holes),
  ≤560px, shown in a frosted-glass card. Generated by `tools/figure.py`.
- Fonts: system stack only (no webfont — al-folio renders the OS font; a webfont read
  as "AI default"). Accent: teal `#1f6357` / dark `#6cba9f` (oxblood→pink was rejected).

## Gotchas (these cost time if you don't know them)
- **Dev cache**: the browser caches `app.js`/`style.css`. When verifying via the
  preview tools, hard-reload by navigating to `…html?cb=<timestamp>` and re-fetch
  assets with `{cache:'reload'}` — otherwise you'll see stale output and chase ghosts.
- **`.wrap` padding trap**: `main` also has class `wrap`, and `.wrap{padding:0 …}`
  (class) beats `main{padding…}` (element). Top spacing must go on `#main` (id) to win.
- **Preview screenshots** time out on this machine (even about:blank). Verify with
  `preview_snapshot` / `preview_eval` (computed styles) instead, not screenshots.
- **Preview viewport** sometimes reports innerWidth 0 → widths read as ~2–35px; resize
  it before trusting layout measurements.
- **PDF figures**: most extract via largest embedded image; some journals slice figures
  into horizontal strips (no single image) → use `figure.py --page-region`.
- **Publisher scraping**: ACS/Cell/Elsevier 403 plain curl; a browser User-Agent gets
  some `og:image`/HTML, but corresponding-author emails are often JS-injected (not in
  static HTML) — ask the user for those.
